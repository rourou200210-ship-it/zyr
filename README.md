# 🧠 AI Emotion Recognition Web Application

## 📌 프로젝트 설명 (Project Description)

본 프로젝트는 생성형 AI를 활용한 개발 과정을 기반으로, Google Teachable Machine을 통해 구축된 경량형 감정 인식 웹 애플리케이션이다. 시스템은 카메라를 통해 사용자의 얼굴 이미지를 수집하고, 표정을 분석하여 **분노(Anger), 슬픔(Sadness), 기쁨(Happiness)**의 세 가지 기본 감정으로 분류한다.

본 프로젝트의 핵심 목적은 저진입 장벽 AI 도구를 활용한 감정 인식의 실현 가능성을 탐색하고, 실제 환경에서의 성능과 한계를 검증하는 데 있다. 전체 개발 과정은 데이터 수집, 모델 학습, 테스트 검증 및 오분류 분석으로 구성된다.

또한 본 시스템은 TensorFlow.js를 활용하여 브라우저 상에서 모델이 직접 실행되는 구조로 설계되었으며, 별도의 서버 없이도 동작하는 경량화 및 높은 배포 편의성을 갖는다.

---

## ⚙️ 기술 구성 (Tech Stack)

- Google Teachable Machine (모델 학습)
- TensorFlow.js (프론트엔드 추론)
- HTML / JavaScript (웹 개발)
- 생성형 AI (코드 생성 및 개발 보조)

---

## 📊 모델 성능 (Model Performance)

- 전체 정확도: 약 “88%”  
- 분노 인식: 약 “92%”
- 슬픔 인식: 약 “85%”
- 기쁨 인식: 약 “80%”

이 모델은 정면 얼굴, 안정적인 조명, 명확한 표정과 같은 이상적인 환경에서는 비교적 높은 성능을 보이지만, 실제 복잡한 환경에서는 성능이 저하될 수 있다.

---

## ⚠️ 한계점 (Limitations)

- 조명 변화에 민감함 (과도하게 밝거나 어두운 환경에서 성능 저하)
- 얼굴 가림(마스크, 손 등)에 의해 정확도 감소
- 데이터 다양성 부족으로 인한 편향 존재
- 인종 및 연령에 따른 일반화 성능 한계

## 🎯 활용 목적 (Use Cases)

본 프로젝트는 “감정 인식 AI의 실험적 프로토타입 시스템”으로서 다음과 같은 활용이 가능하다:

### 1️⃣ 감정 보조 모니터링
카메라를 통해 사용자의 표정을 실시간으로 분석하여 감정 변화 추이를 파악하고, 감정 관리에 참고 자료를 제공한다.

### 2️⃣ 정신 건강 조기 경고
감정이 장기간 비정상적인 패턴을 보일 경우(지속적인 우울 또는 과도한 흥분 등), 간단한 알림을 통해 초기 위험을 감지할 수 있다.

### 3️⃣ 감정 데이터 기록
사용자의 감정 상태를 지속적으로 기록하여 시계열 데이터로 축적하고, 장기적인 분석 및 연구에 활용할 수 있다.

### 4️⃣ 교육 및 연구 활용
다음과 같은 분야에서 활용 가능하다:
- AI 기초 교육
- 인간-컴퓨터 상호작용(HCI) 실험
- 감정 컴퓨팅(Affective Computing) 연구

### 5️⃣ 프로토타입 시스템 개발
저비용으로 빠르게 AI 시스템을 검증할 수 있는 도구로 활용 가능:
- 웹 인터랙션 실험
- 인터페이스 디자인
- AI 제품 초기 테스트
